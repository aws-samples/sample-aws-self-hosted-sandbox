# Fly.io vs AWS Graviton 沙盒成本对比

> 场景：**1000 个 4 vCPU / 8 GiB 沙盒，全部同时持续 active**（730 h/月）。
> 区域：AWS = us-east-1（与 us-west-2 同价，见 §6）；Fly.io = 北美/欧洲（ams 基准）。
> 币种：USD。数据日期：2026-06-30。
> 本文 §1~§5 为**计算成本**；**带宽**为独立大变量，见 §7。

---

## 1. 结论速览（TL;DR）

| 维度 | 结论 |
|---|---|
| 同口径"独占核" | AWS c6g.metal 与 Fly performance-4x 按需相近；**锁长约后 AWS 3 年省约 40%** |
| 同口径"超卖" | Fly shared-cpu-4x 与 AWS c6g.metal 超卖按需相近；**AWS 3 年仍省约 40%** |
| 全局最便宜 | **AWS c6g.metal + 超卖 + 3 年 SP ≈ $15.5K/月**（依赖快照挂起空闲沙盒 + 锁 3 年）|
| Fly 结构性短板 | **Fly 无 3 年承诺档**，只有单一 1 年预留 −40%；AWS 可锁 3 年 −62% |
| 选机型 | AWS 仅考虑 **c6g.metal**（每 vCPU 最便宜的现代 Graviton；metal 无溢价）|
| 选区域 | us-east-1 / us-west-2 **同价**，换区零收益（见 §6）|

## 3. 怎么读这张表

**先对齐口径**——要比就比"是否独占物理核"一致的两行：

- **独占 / 不超卖**：Fly performance-4x vs AWS c6g.metal 不超卖
  - OD：Fly $128.8K **比 AWS $122.3K 还略贵**；
  - 长约后 AWS 完胜：AWS 3 年 $46K vs Fly 预留 $77K，**省约 40%**。
- **超卖**：Fly shared-cpu-4x vs AWS c6g.metal 超卖
  - OD：Fly $43.1K vs AWS $41.3K，接近；
  - 长约后 AWS 3 年 $15.5K vs Fly 预留 $25.8K，**省约 40%**。

**全局最便宜路径**：AWS c6g.metal + 超卖 + 3 年 SP = **$15.5K/月（$15.53/沙盒）**，是 Fly performance 按需（$128.8K）的约 **1/8**。前提：① 沙盒多数 idle、靠快照挂起回收内存（repo 已具备此机制）；② 锁定 3 年。

## 4. ⚠️ 超卖在两个平台的含义完全不同

| | 计费模型 | "超卖"如何省钱 |
|---|---|---|
| **Fly.io** | 按 **machine** 计费，1000 沙盒 = 1000 份，份数跑不掉 | 选 shared 让**单价**从 $128.77 → $43.08；但仍付 1000 份 |
| **AWS c6g.metal** | 按**整台裸机**计费 | 超卖让你用**更少裸机**装下 1000 沙盒（77 → 26 台）|

**关键约束**：4核8G 是 **1:2** 配比，c6g.metal（64 核 / 128 GiB）也是 **1:2**。
→ 不超卖时 CPU 与内存**同时打满**（都是 13 个/台，受双约束）。
→ **单纯超卖 CPU 无效**，因为内存先耗尽（128÷8=16，留 15% 开销 = 13）。要提密度**必须同时超卖内存**，而内存超卖只有配合**快照挂起空闲沙盒释放内存**（本项目 microVM 机制）才安全。

**AWS 超卖行 = 综合超卖 3:1 的假设值**（沙盒多数 idle、平均实占 ~2.7 GiB、靠快照回收空闲），即 39 个/台。可调，见 §5。

## 6. 区域：us-east-1 vs us-west-2 — 同价

经 AWS 官方 Price List API 逐项比对，本场景所有价格维度两区**逐分相同**：
c6g.metal On-Demand / RI 各档、Internet 出口阶梯、NAT Gateway、EKS 控制面、EBS gp3 全部一致。
→ **换区零收益**；选区应看容量可得性 / 延迟 / 合规，而非价格。
（仅 Spot 价格按 AZ 供需浮动，可能有差异，但常驻满载沙盒不适合 Spot。）

## 7. 带宽（egress）— 独立大变量，务必关注

Fly 方案中 AI gateway 和 bucket 在 Fly 内网 → 流量**免费**。迁到 AWS 后关键看流量去向：

| 流量去向 | AWS 成本 | vs Fly 内网免费 |
|---|---|---|
| → 同区域 S3（走 **Gateway Endpoint**）| 🟢 $0/GB | 持平 |
| → 同区域 Bedrock / AWS 服务 | 🟢 $0/GB | 持平 |
| → 公网外部 API，走 **NAT** | 🔴 $0.045 + $0.09 = **$0.135/GB** | 比 Fly $0.02 贵 6.75× |
| → 公网，走 **IPv6 Egress-only IGW** | 🟡 $0.09/GB（无 NAT 费）| 比 Fly 贵 4.5× |

**降本三件套**：① 存储走 S3 Gateway Endpoint（免费）；② AI 调用落 Bedrock 同区域（repo 已是此架构，经集群内 LiteLLM 代理）；③ 必须出公网时用 IPv6 Egress-only IGW 去掉 NAT $0.045/GB 处理费。
做到这三点，可把"出口成本 6.75× 于 Fly"压回接近 Fly 内网的 $0。

> 注：本项目 repo 当前 `enable_nat_gateway=false`，节点公有子网 + IGW 直出，已无 NAT 费。

## 8. 非计算费用对比表（带宽 + 其他费用组成）

> §2 是**计算成本**；本节是**计算之外的全部费用**。核心信息：很多在 Fly 内网免费的项，迁到 AWS 后开始计费。

### 8.1 总览：Fly.io vs AWS 逐项对比（1000 沙盒）

| 费用项 | Fly.io 单价 | Fly 1000沙盒/月 | AWS 单价 | AWS 1000沙盒/月 | 迁移影响 |
|---|---|---|---|---|---|
| **公网出口（北美/欧）** | $0.02/GB | 见 8.2 | 阶梯 $0.09→$0.05/GB | 见 8.2 | 🔴 AWS 贵 4.5–6.75× |
| **内网（AI gateway↔沙盒↔bucket）** | **免费** | **$0** | S3 Endpoint $0 / 跨AZ $0.01 / NAT $0.045 | $0 ~ 视架构 | ⚠️ 看是否走 Endpoint |
| **NAT Gateway** | 无此概念 | $0 | $0.045/h + $0.045/GB | $97（3网关）+ 流量 | 🔴 AWS 独有；可用 IGW 规避 |
| **公网 IPv4 地址** | 共享免费 | $0 | $0.005/h ≈ $3.6/月/个 | 0 ~ $3,600 | 🔴 AWS 新增；用共享入口可避 |
| **入口 LB / Anycast** | 免费（Fly Proxy） | $0 | NLB ~$16/月 + LCU | ~$20–50 | 🟡 AWS 用单个共享 NLB 即可 |
| **TLS 证书** | 前10免费，单域$0.10/通配$1 | ~$0 | ACM 免费 | $0 | 持平 |
| **持久存储** | Volumes $0.15/GB/月 | $1,500（10GB/台） | EBS gp3 $0.08/GB/月 / S3 $0.023/GB | $800（10GB/台） | 🟢 AWS 更便宜 |
| **快照存储** | $0.08/GB/月，前10GB免费 | 视量 | S3 $0.023/GB/月 | 视量 | 🟢 AWS 更便宜 |
| **组织 plan / 强制月费** | $0（纯 PAYG） | $0 | $0 | $0 | 持平 |
| **EKS 控制面** | 无 | $0 | $0.10/h | $73 | 🔴 AWS 独有（单集群固定）|
| **Support（可选）** | $0/$29/$199/$2500+ | $0 | Business 起 ~$100+ 或 3% 账单 | 视选择 | 企业级另算 |

### 8.2 带宽三档（公网出口，1000 沙盒）

| 每沙盒出口/月 | 总量 | **Fly（$0.02/GB）** | **AWS 走 NAT（$0.135/GB）** | **AWS IPv6-IGW（$0.09/GB）** | AWS 走内网/S3E |
|---|---|---|---|---|---|
| 10 GB | 10 TB | $200 | $1,447 | $900 | $0 |
| 50 GB | 50 TB | $1,000 | $6,647 | $4,500 | $0 |
| 200 GB | 200 TB | $4,000 | $22,897 | $18,000 | $0 |

> AWS NAT 列含 NAT 处理费 + Internet 出口阶梯 + 3 个 NAT Gateway 固定费 $97。
> "走内网/S3E"列：若目标是同区域 S3（Gateway Endpoint）或 Bedrock，则 $0——这正是对标 Fly 内网免费的关键。

### 8.3 关键结论：迁移后新增/消失的费用

**Fly 免费、迁 AWS 开始收费的项：**
1. **内网流量**：Fly 同区同组织 6PN 全免费；AWS 跨 AZ $0.01/GB、NAT $0.045/GB、跨区 $0.02/GB。→ 用 **S3 Gateway Endpoint + Bedrock 同区域** 可压回 $0。
2. **公网 IPv4**：Fly 共享免费；AWS 每个 $3.6/月，1000 个独立 IP ≈ $3,600/月。→ 用**共享入口（单 NLB + 通配符域名，repo 已是此架构）**可避免。
3. **入口 LB**：Fly Proxy 免费；AWS NLB 有小时费 + LCU。→ 单个共享 NLB 成本可忽略。
4. **EKS 控制面**：AWS 独有固定 $73/月（与规模无关）。

**AWS 反而更便宜的项：**
- **持久存储**：EBS gp3 $0.08 vs Fly Volumes $0.15（约省 47%）；S3 $0.023 更低。
- **快照**：S3 $0.023 vs Fly $0.08。

**净结论**：非计算费用里，**带宽是唯一可能反转计算成本优势的大头**。只要遵守"内网走 S3 Endpoint + Bedrock、公网走 IPv6-IGW、入口用共享 NLB"，AWS 非计算费用可控制在与 Fly 接近甚至更低的水平；否则（沙盒裸走 NAT + 每个独立 IPv4），单这两项就能多出 $6K–$26K/月。

## 9. 托管服务对比：AI Gateway（→LiteLLM）与 Tigris（→S3）

客户在 Fly.io 上额外用了两个服务：**AI Gateway**（调 LLM）和 **Tigris**（对象存储）。本节对标我们方案的 **LiteLLM+Bedrock** 与 **S3**。

### 9.1 AI Gateway → 自建 LiteLLM + Bedrock

> ⚠️ **前提说明**：经核实（`fly.io/docs` 导航、`fly.io/ai`、pricing 页三处交叉验证，2026-06-30），**Fly.io 官方并无一个名为 "AI Gateway" 的托管 LLM 网关产品**，且 Fly GPU 已于 2025-08 下线。客户所说的 "AI Gateway" 应是**其自行在 Fly 上部署的 LLM 网关**（LiteLLM/OpenRouter 风格的 BYOK 代理）或泛指"调 LLM 那一层"。以下按此解读对标；若客户实指某第三方托管网关（如 OpenRouter/Cloudflare AI Gateway/Portkey），需单独补该产品定价。

**关键结论：对比要拆两层，且大头不在网关本身。**

| 层次 | Fly 侧（自托管网关） | AWS 侧（自建 LiteLLM+Bedrock） | 差异 |
|---|---|---|---|
| **① 网关服务费/算力** | 一台 Machine：shared-cpu-1x/256MB ≈ **$2/月**，或 performance-1x/2GB ≈ **$32/月** | LiteLLM Pod（requests 250m/1Gi）≈ **$11/月**，满载封顶（limits 2C/4Gi）≈ **$73/月** | 两边都是**几美元到几十美元/月**，相对 token 费是零头 |
| **② 底层 LLM token 费** | 透传给背后 provider（BYOK，网关不加价） | 透传给 **AWS Bedrock**（按各模型 per-1M token 计价） | **真正的大头**；两方案都要付，与网关无关 |

**要点：**
- **网关这一层两边都近乎免费**（自建代理就是一个小 Pod / 一台小 Machine 的钱），不构成有意义的成本差异。
- **决定成本的是 token 费**，它是透传成本：我们走 **Bedrock 的 per-token 价**，Fly 侧走其背后 provider 的价。**这不该算作"网关服务"的成本差**，而是模型选型/用量问题，需按实际模型和月 token 量单独测算。
- 若要精确对比 token 费，需要客户提供：**用哪些模型、月均输入/输出 token 量**——给我这些可以拉 Bedrock 各模型单价算一版。

### 9.2 Tigris → AWS S3

数据来源：Tigris 官方定价页（`tigrisdata.com/pricing/`）+ AWS 官方 Price List API（S3 + DataTransfer），可信度高。

| 费用项 | Tigris | AWS S3 (us-east-1) | 差异 |
|---|---|---|---|
| 存储 Standard | **$0.02**/GB/月（全球多区复制内含） | 首 50TB **$0.023** / next 450TB $0.022 /GB/月 | Tigris 省 ~13%，且无跨区复制附加费 |
| Class A 请求（PUT/POST/LIST） | $0.005 / 千次 | $0.005 / 千次 | 相同 |
| Class B 请求（GET/HEAD） | $0.0005 / 千次 | $0.0004 / 千次 | S3 略便宜 20% |
| **公网出口 egress** | **$0（全免，官方明示）** | 阶梯 $0.09→$0.05/GB（首 100GB/月免费） | **核心分水岭** |
| 同云内网访问 | Fly app↔Tigris 免费 | EC2↔S3 走 **Gateway Endpoint 免费** | 都 ≈ $0 |
| 免费额度 | 5GB + 1万 A + 10万 B / 月 | 100GB/月出口 + 新客 6 月 $200 | 千沙盒规模下均可忽略 |

**场景化月成本**（1000 沙盒，假设 5000万 Class A + 5亿 Class B 请求）：

| 存储量 | 出口前提 | Tigris | AWS S3 |
|---|---|---|---|
| 1TB | 同内网（egress≈0） | ~$520 | ~$474 |
| 10TB | 同内网 | ~$705 | ~$686 |
| 50TB | 同内网 | ~$1,524 | ~$1,628 |
| 10TB | **+20TB 公网出口** | **~$705** | **~$2,478** |

**结论：**
1. **egress 是唯一分水岭。** 若存储与计算同云内网（我们 repo 就是 EC2↔S3 走 Gateway Endpoint），两者 egress 都 ≈$0，**基本打平**（存储越大 Tigris 略占优，请求侧 S3 略便宜）。
2. **只要有可观公网出口**（rootfs/快照跨环境拉取、对外分发），Tigris 免费 egress **碾压性便宜**，可省数千美元/月。这是 Tigris 的核心卖点，也和"rootfs/snapshot 常被跨环境拉取"的场景相关。
3. **对我们方案的启示**：坚持 **S3 + Gateway Endpoint + 同区域访问**，把出口压在内网，S3 成本即可与 Tigris 打平；一旦沙盒需大量对公网拉存储，需评估 CloudFront 或接受 egress 费。

## 10. 价格来源与可信度

- **AWS**：官方 Price List API，EC2 offer 版本 **20260626063016**（2026-06-26 快照），us-east-1，SKU `B9CQBH4FXSKWQPFQ`（Linux / Shared / No License）。逐分精确，可信度高。
  - c6g.metal：OD **$2.176/h**；1 年 All-Upfront 有效 **$1.27945/h**；3 年 All-Upfront 有效 **$0.81819/h**（upfront $21,502，recurring $0）。
  - "1 年 / 3 年" 用 Standard RI All-Upfront 作为 Compute Savings Plan 的代表值；实际 Compute SP 通常略贵 2–3%（换取跨实例族/区域灵活性）。
  - metal 无溢价：c6g.metal = c6g.16xlarge = $2.176/h（逐分相同）。
  - 非计算项：NAT Gateway $0.045/h + $0.045/GB、Internet 出口阶梯 $0.09/$0.085/$0.07/$0.05、公网 IPv4 $0.005/h、S3 Gateway Endpoint 免费、同区域 S3↔EC2 免费、跨 AZ $0.01/GB、EBS gp3 $0.08/GB、EKS $0.10/h，均来自 AWS 官方定价页/VPC 定价页，可信度高。
- **Fly.io**：官方定价文档 `fly.io/docs/about/pricing/`，可信度高。
  - shared-cpu-4x（8GB）= 基础价 $8.08（默认 1GB）+ 7GB × $5/GB = **$43.08/月**。
  - performance-4x（含 8GB）= **$128.77/月**。
  - 预留（reservation）：**单一 1 年期，统一 −40%**，预付全年 + 按月发 credit（不滚存）；shared/performance 均适用；credit **仅作用于 CPU+RAM，不含带宽/存储**。**Fly 无 3 年档**。
  - 非计算项：出口 $0.02/$0.04/$0.12/GB（按区）、内网免费、专用 IPv4 $2/月、Volumes $0.15/GB/月、快照 $0.08/GB/月（前 10GB 免费）、TLS 单域 $0.10/通配 $1（前 10 免费）、无组织月费、support 可选。Docker registry 存储官方文档未列费（推定不单收，可信度中）。

## 11. 关键假设（可调，影响结论）

1. 1000 个沙盒**全部同时 active 满载**、730 h/月。
2. AWS 系统开销预留 15%（可用 = 标称 × 0.85）。
3. AWS 超卖行按**综合 3:1**；依赖快照挂起空闲沙盒回收内存。
4. 本表仅计算成本，**未含**带宽、存储、EKS 控制面（$73/月）、EBS。
5. Fly 区域取北美/欧洲基准；亚太/南美计算与带宽更贵。
6. AWS Compute SP 实时折扣上线前请用 AWS Pricing Calculator 复核（±2–3%）。
